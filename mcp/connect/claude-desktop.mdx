## What this enables

Once connected, you can ask Claude things like:

- *"What agents do I have in FluxPrompt?"*
- *"Show me the structure of my Lead Scorer agent"*
- *"What node types are available in FluxPrompt?"*

Claude will call the FluxPrompt MCP tools automatically and show you the results inline.

## Prerequisites

- Claude Desktop installed ([download here](https://claude.ai/download))
- Your FluxPrompt workspace MCP token (see [Authentication](/mcp/authentication))

## Step-by-step

<Steps>
  <Step title="Find the Claude Desktop config file">
    The config file location depends on your operating system:

    | OS | Path |
    |----|------|
    | **macOS** | `~/Library/Application Support/Claude/claude_desktop_config.json` |
    | **Windows** | `%APPDATA%\Claude\claude_desktop_config.json` |
    | **Linux** | `~/.config/Claude/claude_desktop_config.json` |

    If the file doesn't exist yet, create it.
  </Step>
  <Step title="Add the FluxPrompt MCP server">
    Open the file in any text editor and add (or merge) the following:

    ```json
    {
      "mcpServers": {
        "fluxprompt": {
          "url": "https://mcp.fluxprompt.ai/mcp",
          "headers": {
            "Authorization": "Bearer YOUR_MCP_TOKEN"
          }
        }
      }
    }
    ```

    Replace `YOUR_MCP_TOKEN` with your actual token (starts with `mcp_`).

    <Warning>
    If the file already has an `mcpServers` key, add `"fluxprompt": { ... }` inside it — do not create a second `mcpServers` block.
    </Warning>
  </Step>
  <Step title="Restart Claude Desktop">
    Fully quit Claude Desktop (not just close the window) and reopen it.

    - **macOS:** right-click the Claude icon in the Dock → Quit
    - **Windows:** right-click the Claude icon in the system tray → Exit
  </Step>
  <Step title="Verify the connection">
    In a new conversation, type:

    > *List my FluxPrompt agents*

    Claude should respond with a list of agents from your workspace. If you see an error, check the [Troubleshooting](/mcp/troubleshooting) page.
  </Step>
</Steps>

## Full config example

If your `claude_desktop_config.json` is empty or new:

```json
{
  "mcpServers": {
    "fluxprompt": {
      "url": "https://mcp.fluxprompt.ai/mcp",
      "headers": {
        "Authorization": "Bearer mcp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

If you already have other MCP servers configured:

```json
{
  "mcpServers": {
    "some-other-server": {
      "command": "npx",
      "args": ["-y", "@some/server"]
    },
    "fluxprompt": {
      "url": "https://mcp.fluxprompt.ai/mcp",
      "headers": {
        "Authorization": "Bearer mcp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
      }
    }
  }
}
```

## What Claude can do with FluxPrompt

After connecting, Claude has access to these tools:

| Tool | What it does |
|------|-------------|
| `list_agents` | Lists all agents in your workspace |
| `get_agent` | Fetches the full structure of a specific agent |
| `list_node_types` | Lists all available FluxPrompt node types |

Claude will choose the right tool automatically based on your question. You can also ask it explicitly: *"Use `get_agent` to show me the structure of agent ID abc123"*.

## Troubleshooting

If the connection isn't working, see the [Troubleshooting](/mcp/troubleshooting) page. The most common issues are:

- Typo in the token (check for extra spaces)
- Not fully quitting Claude Desktop before restarting
- JSON syntax error in the config file (missing comma, extra brace)
