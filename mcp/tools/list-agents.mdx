title: "list_agents"
description: "List all agents in your FluxPrompt workspace."

## Description

`list_agents` returns every agent (flow) in your workspace that your token has access to. The response includes each agent's ID, name, description, node count, and public URL — everything you need to reference or inspect an agent.

## Input

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `search` | string | No | Case-insensitive substring filter on the agent name. Omit to return all agents. |
| `cursor` | string | No | Opaque pagination cursor returned as `nextCursor` by a previous call. Omit to fetch the first page. Treat the value as opaque — do not parse, construct, or reuse it across tools. |

No parameters are required. Calling `list_agents` with no input returns the first page of all agents in your workspace. The server controls the page size.

## Output

```json
{
  "agents": [
    {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "name": "Lead Scorer",
      "description": "Scores inbound leads based on company size and tech stack",
      "updatedAt": "2026-05-31T14:22:00Z",
      "nodeCount": 12,
      "isPublic": false
    },
    {
      "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
      "name": "Invoice Parser",
      "description": null,
      "updatedAt": "2026-06-01T09:45:00Z",
      "nodeCount": 7,
      "isPublic": true
    }
  ],
  "total": 19,
  "nextCursor": "eyJvIjo1MH0"
}
```

### Output fields

| Field | Type | Description |
|-------|------|-------------|
| `agents` | array | List of agents for this page. |
| `agents[].id` | string (UUID) | Unique identifier. Use this with `get_agent`. |
| `agents[].name` | string | Agent display name as shown in the FluxPrompt UI. |
| `agents[].description` | string \| null | Agent description, or `null` if none is set. |
| `agents[].updatedAt` | ISO 8601 string | When the agent was last modified. |
| `agents[].nodeCount` | number | Number of nodes in the agent's current version. |
| `agents[].isPublic` | boolean | Whether the agent has a public-facing endpoint. |
| `total` | number | Total number of agents matching the query. Informative — use `nextCursor` to drive pagination. |
| `nextCursor` | string | Opaque cursor for the next page. **Present only when more results exist; omitted on the last page.** |

## Pagination

`list_agents` uses opaque-cursor pagination ([MCP 2025-11-25](https://modelcontextprotocol.io)). The server decides the page size. To page through results, call again passing the previous `nextCursor` as `cursor`, and stop when `nextCursor` is absent:

```
Call 1: list_agents({})                        → page 1, nextCursor = "eyJvIjo1MH0"
Call 2: list_agents({ cursor: "eyJvIjo1MH0" }) → page 2, nextCursor = "eyJvIjoxMDB9"
Call 3: list_agents({ cursor: "eyJvIjoxMDB9" })→ page 3, no nextCursor → last page
```

<Warning>
Cursor values are opaque. Do not parse or construct them, and never pass a cursor returned by a different tool. A malformed or invalid cursor returns JSON-RPC error `-32602` (Invalid params) with `data: { "invalid": "cursor" }`.
</Warning>

## Example prompts

You can ask your AI client in plain language — it will call `list_agents` automatically:

- *"Show me all my FluxPrompt agents"*
- *"List the agents in FluxPrompt and tell me which ones are public"*
- *"How many agents do I have in FluxPrompt?"*
