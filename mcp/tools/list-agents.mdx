title: "list_agents"
description: "List all agents in your FluxPrompt workspace."
---

## Description

`list_agents` returns every agent (flow) in your workspace that your token has access to. The response includes each agent's ID, name, description, node count, and public URL — everything you need to reference or inspect an agent.

## Input

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `search` | string | No | Case-insensitive substring filter on the agent name. Omit to return all agents. |
| `limit` | number | No | Maximum number of agents to return. Default: 20. Maximum: 200. |
| `offset` | number | No | Number of agents to skip, for pagination. Default: 0. |

No parameters are required. Calling `list_agents` with no input returns the first page of all agents in your workspace.

## Output

```json
{
  "agents": [
    {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "name": "Lead Scorer",
      "description": "Scores inbound leads based on company size and tech stack",
      "updatedAt": "2026-05-31T14:22:00Z",
      "nodeCount": 12,
      "isPublic": false
    },
    {
      "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
      "name": "Invoice Parser",
      "description": null,
      "updatedAt": "2026-06-01T09:45:00Z",
      "nodeCount": 7,
      "isPublic": true
    }
  ],
  "total": 19
}
```

### Output fields

| Field | Type | Description |
|-------|------|-------------|
| `agents` | array | List of agents. |
| `agents[].id` | string (UUID) | Unique identifier. Use this with `get_agent`. |
| `agents[].name` | string | Agent display name as shown in the FluxPrompt UI. |
| `agents[].description` | string \| null | Agent description, or `null` if none is set. |
| `agents[].updatedAt` | ISO 8601 string | When the agent was last modified. |
| `agents[].nodeCount` | number | Number of nodes in the agent's current version. |
| `agents[].isPublic` | boolean | Whether the agent has a public-facing endpoint. |
| `total` | number | Total number of agents matching the query (ignoring `limit`/`offset`), so you know how many pages remain. |

## Pagination

`list_agents` uses offset-based pagination. Compare `total` against `limit` + `offset` to know whether more agents remain, then advance `offset`:

```
Call 1: list_agents({ limit: 20 })              → agents 1-20,  total = 45
Call 2: list_agents({ limit: 20, offset: 20 })  → agents 21-40, total = 45
Call 3: list_agents({ limit: 20, offset: 40 })  → agents 41-45, total = 45
```

## Example prompts

You can ask your AI client in plain language — it will call `list_agents` automatically:

- *"Show me all my FluxPrompt agents"*
- *"List the agents in FluxPrompt and tell me which ones are public"*
- *"How many agents do I have in FluxPrompt?"*

## Next steps

Once you have an agent's `id`, use [`get_agent`](/mcp/tools/get-agent) to fetch its full node and edge structure.
