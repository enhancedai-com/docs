## Lista de verificação de conexão

Antes de investigar erros específicos, verifique estas cinco coisas:

<Steps>
  <Step title="O método de autenticação combina com o cliente">
    Claude Desktop e Claude.ai usam **somente OAuth** — sem token. Se você está tentando colar um token neles, esse é o problema; adicione o FluxPrompt como conector personalizado. Os passos abaixo valem para clientes que usam service token (Cursor, n8n, CLI, serviços de backend).
  </Step>
  <Step title="Formato do token (clientes com service token)">
    Seu service token deve começar com `mcp_` e não conter espaços. Se você o copiou de **Settings → MCP Tokens**, verifique se há espaços em branco no final.
  </Step>
  <Step title="Formato do header Authorization (clientes com service token)">
    Deve ser `Bearer <token>` (B maiúsculo, um espaço). Não `bearer`, não `Token`, não o token puro sem o prefixo.

    ✅ `Bearer mcp_abc123`
    ❌ `mcp_abc123`
    ❌ `bearer mcp_abc123`
  </Step>
  <Step title="URL do servidor">
    A URL deve ser `https://mcp.fluxprompt.ai/mcp` (sem barra no final). Se a sua equipe usa um endpoint personalizado, confirme a URL com o FluxPrompt.
  </Step>
  <Step title="Reinicialização do cliente">
    Para o Claude Desktop: feche completamente (bandeja do sistema → Sair) e reabra — fechar a janela não é suficiente.
  </Step>
  <Step title="Sintaxe JSON">
    Os arquivos de configuração (claude_desktop_config.json, cursor/mcp.json) devem ser JSON válido. Use [jsonlint.com](https://jsonlint.com) para validar se você editou manualmente.
  </Step>
</Steps>

## Referência de erros

### O Claude Desktop diz que o token é inválido / não consigo colar meu token

**Causa:** O Claude Desktop (e o Claude.ai) rejeitam tokens bearer colados pelo usuário. Eles autenticam via **OAuth**, não por um token em arquivo de configuração.

**Solução:** Não cole um token. Adicione o FluxPrompt como conector personalizado e conclua o fluxo de login + consentimento — veja [Conectar Claude Desktop](/mcp/pt-br/connect/claude-desktop). Service tokens são apenas para clientes como Cursor, n8n, a CLI e serviços de backend.

---

### "Unauthorized" / 401 (clientes com service token)

**Causa:** Para clientes baseados em token (Cursor, n8n, CLI, serviços de backend), o service token está ausente, malformado ou revogado. (Claude Desktop / Claude.ai não usam token — veja a entrada acima.)

**Solução:**
1. Confirme que o seu header `Authorization` é `Bearer mcp_xxxxx` sem caracteres extras.
2. Verifique se o token ainda está ativo em **Settings → MCP Tokens**, ou gere um novo.

---

### "Agent not found" (`isError: true`)

**Causa:** O `agentId` passado não existe no seu workspace, ou pertence a um workspace diferente. O `get_agent` reporta isso como erro de tool (`result.isError: true`), e não como um código de erro de protocolo.

**Solução:**
1. Use `list_agents` para obter IDs válidos do seu workspace.
2. Confirme que você está usando o token de workspace correto (se tiver tokens para múltiplos workspaces).

---

### As tools retornam lista vazia (`"agents": []`)

**Causa:** Seu workspace não tem agentes, ou o token não tem acesso ao workspace.

**Solução:**
1. Faça login no FluxPrompt e confirme que você tem pelo menos um agente.
2. Se tiver, entre em contato com o FluxPrompt — seu token pode estar com escopo no workspace errado.

---

### "Method not found" / -32601

**Causa:** O nome da tool está errado (ex.: `listAgents` em vez de `list_agents`).

**Solução:** Use os nomes exatos das tools com underscores: `list_agents`, `get_agent`, `list_node_types`.

---

### "Service disabled" / -32005

**Causa:** O servidor FluxPrompt MCP está temporariamente desabilitado para manutenção.

**Solução:** Aguarde alguns minutos e tente novamente. Verifique a [página de status do FluxPrompt](https://status.fluxprompt.ai) para incidentes em andamento.

---

### Claude Desktop não mostra tools MCP

**Causa:** O arquivo de configuração não foi lido porque o Claude Desktop não foi totalmente reiniciado, ou o JSON está malformado.

**Solução:**
1. Feche completamente o Claude Desktop (clique com o botão direito na bandeja do sistema → Sair no Windows, ou Dock → Sair no macOS).
2. Valide seu `claude_desktop_config.json` em [jsonlint.com](https://jsonlint.com).
3. Reabra o Claude Desktop e inicie uma nova conversa.

---

### n8n: "Connection refused" ou timeout

**Causa:** A versão do n8n é inferior a 1.22 (sem suporte a Streamable HTTP), ou um problema de rede/firewall.

**Solução:**
1. Verifique sua versão do n8n: **Settings → About**. Atualize para 1.22+ se necessário.
2. A partir da sua instância n8n, verifique se ela consegue acessar `https://mcp.fluxprompt.ai` (nenhum firewall corporativo bloqueando HTTPS de saída).

---

## Ainda com problemas?

Entre em contato com a equipe FluxPrompt com:

1. A mensagem de erro exata ou a resposta JSON de erro
2. Qual cliente você está usando (Claude Desktop / Cursor / n8n / outro)
3. Seu ID de workspace (encontre-o no FluxPrompt em **Settings → Workspace**)

**Não** inclua seu token MCP em nenhuma mensagem de suporte.
