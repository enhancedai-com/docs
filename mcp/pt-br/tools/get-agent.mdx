## Descrição

`get_agent` retorna a definição completa de um agente: seus metadados mais os arrays completos de `nodes` e `edges` que descrevem a lógica do agente. Use isso para inspecionar como um agente foi construído, auditar sua configuração ou entender suas entradas e saídas.

## Entrada

| Parâmetro | Tipo | Obrigatório | Descrição |
|-----------|------|-------------|-----------|
| `agentId` | string (UUID) | Sim | O identificador único do agente. Obtenha este valor a partir de `list_agents`. |
| `includeNodes` | boolean | Não | Se deve incluir os arrays `nodes` e `edges`. Padrão: `true`. Defina como `false` para uma resposta leve, apenas com metadados. |

      "type": "globalStarter",

Quando `includeNodes` é `false`, os arrays `nodes` e `edges` são omitidos e apenas os campos de metadados são retornados.

### Campos da saída

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `id` | string | UUID do agente. |
| `name` | string | Nome de exibição do agente. |
| `description` | string \| null | Descrição do agente. |
| `updatedAt` | string ISO 8601 | Hora da última modificação. |
| `nodeCount` | number | Número de nodes na versão atual do agente. |
| `isPublic` | boolean | Se o agente tem um endpoint público. |
| `nodes` | array | Todos os nodes do agente (omitido quando `includeNodes` é `false`). Veja abaixo. |
| `edges` | array | Todas as conexões entre os nodes (omitido quando `includeNodes` é `false`). |

#### Campos de node

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `id` | string | ID do node (único dentro deste agente). |
| `type` | string | Tipo do node (ex.: `globalStarter`, `commandTextNode`, `conditionNode`). Veja `list_node_types` para a lista completa. |
| `data` | object | Configuração e dados de exibição específicos do node (inclui o `label` do node). O formato depende do `type`. |
| `position` | object | Coordenadas `{ x, y }` no canvas. |
| `width` | number | Largura do node no canvas. |
| `height` | number | Altura do node no canvas. |

#### Campos de edge

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `id` | string | ID do edge. |
| `source` | string | ID do node de origem. |
| `sourceHandle` | string | Qual handle de saída no node de origem este edge parte. |
| `target` | string | ID do node de destino. |
| `targetHandle` | string | Qual handle de entrada no node de destino este edge se conecta. |
| `type` | string | Tipo de renderização do edge (ex.: `custom`). |
| `style` | object | Estilo visual (cor e espessura do traço). |
| `animated` | boolean | Se o edge é animado no canvas. |
| `markerEnd` | object | Configuração da ponta da seta. |

## Erro: agente não encontrado

Se o `agentId` não existir no seu workspace, a chamada retorna um **erro de tool** (não um erro de protocolo) — uma resposta JSON-RPC bem-sucedida cujo `result` tem `isError: true`:

```json
{
  "result": {
    "content": [
      { "type": "text", "text": "Agent \"a1b2c3d4-...\" not found in this workspace." }
    ],
    "isError": true
  }
}
```

Isso também é retornado se o agente existir mas pertencer a um workspace diferente. Por segurança, a resposta não revela se o agente existe fora do seu workspace.

## Exemplos de prompts

- *"Mostre a estrutura completa do agente a1b2c3d4"*
- *"Quais nodes meu agente Lead Scorer usa?"*
- *"Busque o agente com ID a1b2c3d4 e explique como ele funciona"*
