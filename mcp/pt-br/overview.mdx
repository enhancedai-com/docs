## O que é o FluxPrompt MCP?

**FluxPrompt MCP** é um servidor [Model Context Protocol](https://modelcontextprotocol.io) de entrada. Ele permite que clientes de IA externos — Claude Desktop, Cursor, n8n e outros — se conectem ao seu workspace FluxPrompt e interajam com seus agentes por linguagem natural.

Em vez de navegar pela interface do FluxPrompt, seu cliente de IA pode fazer perguntas como:

- *"Liste todos os agentes do meu workspace"*
- *"Mostre os nodes dentro do agente Invoice Parser"*
- *"Quais tipos de node o FluxPrompt suporta?"*

## Autenticação em resumo

O FluxPrompt MCP suporta **dois** métodos de autenticação, e o correto depende do seu cliente:

- **OAuth 2.0** — você adiciona a URL do servidor como conector, faz login por uma tela de consentimento do FluxPrompt e escolhe um workspace. Nenhum token é colado. É **obrigatório** para Claude Desktop e Claude.ai (que rejeitam tokens colados) e também é suportado por Claude Code / CLI e Cursor.
- **Service token estático** — o dono do workspace gera um token (começa com `mcp_`) em [**Settings → MCP Tokens**](https://app.fluxprompt.ai/dashboard-beta?settings=mcpTokens) e o coloca no header `Authorization`. É para clientes máquina-a-máquina, a CLI, o Cursor e o n8n — clientes que não podem ou não devem executar um fluxo de consentimento no navegador.

| Cliente | OAuth | Service token estático |
|---------|-------|------------------------|
| Claude Desktop | ✅ obrigatório | ❌ não suportado |
| Claude.ai | ✅ obrigatório | ❌ não suportado |
| Claude Code / CLI | ✅ | ✅ |
| Cursor | ✅ | ✅ |
| n8n | parcial | ✅ recomendado |
| Serviços de backend (M2M) | ❌ (sem navegador) | ✅ |

Veja [Autenticação](/mcp/pt-br/authentication) para todos os detalhes.

## Como funciona

```
Seu cliente de IA (Claude Desktop / Cursor / n8n)
        │
        │  JSON-RPC sobre HTTPS
        │  Sessão OAuth  ou  Authorization: Bearer <service token>
        ▼
Servidor FluxPrompt MCP  (mcp.fluxprompt.ai)
        │
        │  lê seu workspace
        ▼
FluxPrompt API  ──▶  Seus agentes, nodes e flows
```

1. Você se autentica via **OAuth** (consentimento no navegador) ou um **service token com escopo de workspace**, dependendo do seu cliente.
2. Seu cliente de IA se conecta a `https://mcp.fluxprompt.ai/mcp`.
3. Ele chama as ferramentas — `list_agents`, `get_agent`, `list_node_types` — e recebe respostas estruturadas.
4. Cada chamada é registrada no log de auditoria do FluxPrompt.

## Glossário rápido

| Termo | Significado |
|-------|-------------|
| **MCP** | Model Context Protocol — um padrão aberto para conectar clientes de IA a ferramentas e dados externos. |
| **OAuth** | Um fluxo de consentimento no navegador em que você faz login no FluxPrompt e concede a um cliente acesso a um workspace. Nenhum token é colado. Obrigatório para Claude Desktop e Claude.ai. |
| **Service token** | Uma string secreta com escopo limitado ao seu workspace (começa com `mcp_`), gerada em [**Settings → MCP Tokens**](https://app.fluxprompt.ai/dashboard-beta?settings=mcpTokens). Enviada no header `Authorization`. Usada por clientes máquina-a-máquina, a CLI e o n8n. Diferente da sua API key do FluxPrompt. |
| **Tool** | Uma ação nomeada que o servidor MCP expõe (ex.: `list_agents`). Seu cliente de IA chama as tools em seu nome. |
| **Workspace** | O contexto da sua empresa ou equipe no FluxPrompt. Um token MCP = um workspace. |
| **Streamable HTTP** | O protocolo de transporte que o FluxPrompt MCP utiliza. Um único endpoint HTTPS que aceita requisições JSON-RPC 2.0. |

## Antes de começar

Você vai precisar de:

- Uma **conta FluxPrompt** com pelo menos um agente no seu workspace.
- Um cliente compatível com OAuth (Claude Desktop, Claude.ai, Claude Code, Cursor) ou um **service token** de [**Settings → MCP Tokens**](https://app.fluxprompt.ai/dashboard-beta?settings=mcpTokens) para clientes baseados em token (n8n, serviços de backend).
- Um dos clientes de IA suportados instalado.

<Note>
Este guia assume que o servidor FluxPrompt MCP está em `https://mcp.fluxprompt.ai/mcp`. Se a sua equipe usa um endpoint dedicado, utilize esse no lugar.
</Note>
